Add to a list

This pattern lets the user add information with more than one item to a list.

When to use

Use this pattern when the user needs to add information with more than one item to a list. For example, when they need to enter more than one address or more than one personâ€™s details.

When not to use

Do not use this pattern when the user needs to add different kinds of information that do not relate to each other.

How it works

You need HMRC Frontend to use this pattern â€” find out how to install HMRC Frontend. If you use the HMRC Nunjucks macro, you must copy and paste the entire macro into your code.

Allow the user to:

  • view a summary of what they have added to the list
  • add things to the list
  • check their answers
  • change things on the list
  • remove things from the list
